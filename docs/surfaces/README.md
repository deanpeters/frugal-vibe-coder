# Learning Surfaces

A learning surface is the environment where you build. This repo supports three. They are not a hierarchy — no surface is more advanced or more legitimate than another. They are different tools for different situations and different learners.

---

## Before you choose

Make sure the basics are in place first:

- easiest route: run the guided setup script for your platform from the repo root
- manual route: start in [docs/setup](../setup/)

If you're brand new, the lowest-friction first path is usually:
- install Ollama
- start with Dyad
- move to OpenCode or VS Code only when you want more visibility or more direct control

---

## The three surfaces

| Surface | Tool | Best for |
|--------|------|---------|
| [No-code](no-code.md) | Dyad | First success, rapid prototyping, visual thinkers, low friction |
| [CLI](cli.md) | OpenCode | Transparency, control, understanding what's happening under the hood |
| [IDE](ide.md) | VS Code | Code inspection, real iteration, building transferable skills |

---

## How to choose

**Start with no-code if:**
- you want a working result as fast as possible
- you're more comfortable describing what you want than writing instructions
- you're early in learning and want a confidence-building win

**Start with CLI if:**
- you want to see what the AI is actually doing at each step
- you're comfortable in a terminal and want more control
- you want to understand the process, not just the output

**Start with IDE if:**
- you want to work directly with code files
- you're ready to read, modify, and learn from generated output
- you're building toward longer-term skills

**Not sure?** Start with no-code. You can always move to CLI or IDE once you have a feel for the problem.

---

## The same project, three ways

Each surface guide uses the same example project: a **decision helper** — a simple tool that helps you frame a product decision by organizing what you know, what you don't know, and what questions to ask next.

It's small enough to build quickly on any surface. It's useful enough to actually want. And seeing it built three different ways shows you what each surface is good for and where its limits are.

---

## Moving between surfaces

You're not locked in. Many learners start with no-code, move to CLI when they want more visibility, and eventually use the IDE when they want to inspect and modify what was generated.

Each guide ends with a note on when it makes sense to move to a different surface.

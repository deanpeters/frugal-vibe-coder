# What is Version Control?

## The problem it solves

You're working on something. You make a change. It breaks. You try to undo it, but you can't remember exactly what you changed or what it looked like before.

Or: you want to try something experimental but you're afraid to touch what's already working.

Or: you want to share what you've built with someone else, or pick up where you left off on a different computer.

Version control solves all of these problems.

## What it is

Version control is a system that tracks every change you make to your files over time. It keeps a full history of what changed, when, and why — so you can always go back, compare versions, or share your work with others.

Think of it as a save system that never forgets.

## Git and GitHub — what's the difference

These two names come up together constantly. They are different things.

**Git** is the tool that runs on your computer and tracks changes locally. It's free, open-source, and works entirely offline.

**GitHub** is a website where you can store a copy of your work online. It's free for personal use and makes it easy to access your work from anywhere, share it with others, or use it as a portfolio.

You use git on your machine. You push to GitHub when you want a backup or want to share.

## Why it matters for your learning path

Even if you're not writing code, version control is worth understanding because:

- **Safety net** — you can experiment freely, knowing you can always go back to what worked
- **History** — you can see exactly what changed and when
- **Collaboration** — it's how almost every team shares and coordinates work
- **This repo uses it** — the scripts and docs you're working with are stored in a git repository on GitHub

You don't need to become a git expert. You need to know enough to save your work and not lose it.

## What a repository is

A **repository** (or repo) is just a folder that git is tracking. When you clone this repo to your machine, you get a local copy of everything in it. When you push changes to GitHub, your online copy is updated.

This repo — `frugal-vibe-coder` — is a repository. It lives at github.com/deanpeters/frugal-vibe-coder.

## What this unlocks

Once you have a free GitHub account and understand the basics of git, you can:

- save your work safely and access it from anywhere
- follow setup guides that involve cloning a repo
- track your own projects as they grow
- show others what you've built
